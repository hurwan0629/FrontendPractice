# 05. 옵셔널 프로퍼티와 타입 추론

이번 단계 목표는 세 가지입니다.

1. "있을 수도 없을 수도 있는 값"을 타입으로 표현하기
2. 기본값과 함께 안전하게 다루기
3. TypeScript의 타입 추론을 이해하기

## 1) 옵셔널 프로퍼티란

객체의 어떤 값은 항상 있는 게 아니라, 선택적으로 들어올 수 있습니다.

```ts
type User = {
  id: number;
  name: string;
  nickname?: string;
};
```

`nickname?: string`은 `nickname`이 있을 수도 있고 없을 수도 있다는 뜻입니다.

이 패턴은 React props, API 응답, 폼 데이터에서 매우 자주 나옵니다.

## 2) 왜 필요한가

예를 들어 어떤 사용자는 닉네임이 있고, 어떤 사용자는 없을 수 있습니다.

옵셔널 표시가 없으면 모든 객체에 그 값이 반드시 있어야 합니다.

## 3) 함수와 기본값

옵셔널 값은 그대로 쓰면 위험할 수 있어서, 기본값을 같이 두는 경우가 많습니다.

```ts
function getDisplayName(name: string, nickname?: string): string {
  return nickname ?? name;
}
```

`nickname`이 없으면 `name`을 대신 씁니다.

## 4) 타입 추론

TypeScript는 코드를 보고 타입을 자동으로 추론합니다.

```ts
const count = 10;
```

위 코드는 `count`를 `number`로 추론합니다.

```ts
const user = {
  id: 1,
  name: "Haru",
};
```

이 경우 객체의 각 프로퍼티 타입도 함께 추론합니다.

## 5) 언제 타입을 직접 쓰고, 언제 추론에 맡길까

초반 기준으로는 이렇게 가면 됩니다.

- 함수 파라미터와 반환값은 직접 적기
- 재사용할 객체 구조는 `type`으로 빼기
- 단순한 지역 변수는 추론에 맡기기
