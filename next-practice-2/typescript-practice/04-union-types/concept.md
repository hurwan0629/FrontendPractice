# 04. 유니온 타입과 리터럴 타입

이번 단계 목표는 두 가지입니다.

1. 값이 여러 타입 중 하나일 수 있다는 것을 표현하기
2. 허용된 문자열 값을 제한해서 더 안전한 상태값 만들기

## 1) 유니온 타입이란

유니온 타입은 "이 값은 A이거나 B다"를 표현합니다.

```ts
let age: number | string = 20;
age = "20";
```

위 코드는 `age`가 숫자일 수도 있고 문자열일 수도 있다는 뜻입니다.

실무에서 자주 쓰는 이유는 입력값이나 상태값이 한 종류로만 고정되지 않는 경우가 많기 때문입니다.

## 2) 문자열 리터럴 타입

문자열 전체가 아니라, 특정 문자열만 허용할 수도 있습니다.

```ts
type Status = "loading" | "success" | "error";
```

이제 `Status` 타입에는 세 값만 들어갈 수 있습니다.

```ts
let status: Status = "loading";
```

이 방식은 상태관리, API 요청 상태, 탭 이름, 정렬 기준 같은 데서 매우 유용합니다.

## 3) 유니온 타입을 함수 인자에 쓰기

```ts
function printId(id: number | string): string {
  return `ID: ${id}`;
}
```

숫자 ID를 받을 수도 있고 문자열 ID를 받을 수도 있습니다.

## 4) 왜 좋은가

아무 문자열이나 받게 두는 것보다, 가능한 값의 범위를 좁히면 코드가 훨씬 안전해집니다.

예를 들어:

```ts
type Role = "admin" | "user";
```

이렇게 해두면 `"admim"` 같은 오타를 TypeScript가 잡아줍니다.

## 5) 체크 포인트

이번 단계를 끝내면 아래를 혼자 할 수 있어야 합니다.

- `number | string` 같은 유니온 타입 쓰기
- `"loading" | "success" | "error"` 같은 상태 타입 만들기
- 함수 인자에 유니온 타입 붙이기
- 허용되지 않은 문자열이 왜 에러인지 설명하기
